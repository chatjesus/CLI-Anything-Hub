全自动 7 阶段 流水线

一条命令，从源码分析到 PyPI 发布，全程自动化

1

🔍 分析

扫描源码，映射 GUI 操作到底层 API，理解软件架构与能力边界。

2

📐 设计

架构命令组、状态模型、JSON 输出格式，确保接口一致性与可扩展性。

3

🔨 实现

构建 Click CLI，含 REPL 会话、JSON 输出、撤销/重做，代码质量生产级。

4

📋 规划测试

生成 TEST.md，涵盖单元测试 + E2E 测试计划，包含真实软件调用验证。

5

🧪 编写测试

实现完整测试套件，魔术字节校验、ZIP 结构验证、音频 RMS 检测，绝不信任 exit(0)。

6

📝 文档

更新 TEST.md 记录测试结果，生成应用专属 SOP 文档，供 Agent 自主发现能力。

7

📦 发布

创建 setup.py，pip install -e . 安装到 PATH，cli-anything-软件名 即可使用。

# 用 Python API 直接批量生成报告 → 导出 PDF
from cli_anything.inkscape.core import document, shapes, text, export

# 新建 A4 海报文档
proj = document.create_document("poster", width=1200, height=675)

# 批量添加元素
shapes.add_rect(proj, x=0, y=0, width=1200, height=675, style="fill:#0d0d1a")
shapes.add_circle(proj, cx=900, cy=337, r=200, style="fill:#00d4ff;opacity:0.1")
text.add_text(proj, text="CLI-Anything", x=100, y=240, font_size=96, fill="#ffffff")

# 保存项目 & 用 Inkscape 真实渲染为 PNG
document.save_document(proj, "poster.json")
export.export_svg(proj, "poster.svg")

from cli_anything.inkscape.utils.inkscape_backend import export_svg_to_png
export_svg_to_png("poster.svg", "poster.png", width=1200, height=675)
# → poster.png 89KB, 1200×675, Inkscape 渲染
# REPL 交互模式（所有修改自动共享会话）
$ cli-anything-inkscape

inkscape> document new --width 1200 --height 675 -o poster.json
inkscape> shape add-rect --x 0 --y 0 -w 1200 -h 675 -s "fill:#0d0d1a"
inkscape> text add -t "CLI-Anything" --x 100 --y 240 --font-size 96 --fill "#ffffff"
inkscape> document save
inkscape> export svg poster.svg --overwrite
inkscape> exit

# LibreOffice — 生成 PDF 报告
$ cli-anything-libreoffice document new -o report.json --type writer
$ cli-anything-libreoffice --project report.json writer add-heading -t "Q1 报告" --level 1
$ cli-anything-libreoffice --project report.json export render output.pdf -p pdf
# 与 AI Agent 对接：--json 返回结构化数据
import subprocess, json

def agent_query(cmd):
    result = subprocess.run(
        ["cli-anything-inkscape", "--project", "design.json",
         "--json"] + cmd,
        capture_output=True, text=True
    )
    return json.loads(result.stdout)

# Agent 读取文档当前状态
state = agent_query(["document", "info"])
# → {"name":"design","counts":{"objects":12,"layers":3},...}

# Agent 决策后执行操作
if state["counts"]["objects"] < 5:
    agent_query(["shape", "add-circle",
                  "--cx", "600", "--cy", "337", "--r", "100"])

# --help 让 Agent 自主发现所有能力
caps = subprocess.run(["cli-anything-inkscape", "--help"],
                       capture_output=True, text=True).stdout

四层 测试体系

不信任 exit(0)，每一层都验证真实输出

🔬

Layer 1 · 单元测试

用合成数据隔离测试每个核心函数，覆盖边界条件、参数校验、异常路径。

test_core.py → 项目创建、图层操作、滤镜参数、样式解析
📁

Layer 2 · E2E 原生测试

测试项目文件生成流水线：验证文件格式合法性，不调用真实软件。

验证 ODF ZIP 结构 · MLT XML 合法性 · SVG 标签完整性
🖥️

Layer 3 · E2E 真实后端

调用真实软件执行并验证输出产物，检查魔术字节、像素值、音频 RMS。

LibreOffice→PDF 验证 %PDF- · Blender→PNG 像素分析 · sox 音频时长
⌨️

Layer 4 · CLI 子进程测试

以 subprocess.run 方式调用已安装命令，验证 JSON 输出格式与返回码。

⚠ 关键原则：绝不降级

后端缺失时测试直接 FAIL，而不是 SKIP。确保 CLI 功能的真实性，防止假通过蒙混过关。

⚠ 输出验证规则

PDF 验证 %PDF- 魔术字节；ZIP/OOXML 验证目录结构；PNG 抽样像素值；音频检测 RMS 电平。

⚠ 时码精度陷阱

非整数帧率（29.97fps）累积误差用 round() 而非 int()，测试允许 ±1 帧容差。

⚠ 渲染间隙问题

GUI 软件在渲染时才应用效果，CLI 生成项目文件后必须调用真实渲染器，不能用 Pillow 替代 GIMP。

迭代优化：/refine 命令

广度优化

自动分析当前 CLI 与软件全部能力之间的差距，识别未覆盖的功能模块，增量实现新命令。

/refine ./gimp

聚焦优化

指定特定功能领域进行深度覆盖，精准补充目标场景的命令、测试和文档。

/refine ./gimp "批量图像处理与滤镜"

非破坏性

每次 refine 都是增量操作，不会删除已有命令，可多次运行，持续提升覆盖率。

可反复运行 → 逐步达到 100% 覆盖

自动更新测试

新增命令自动补充对应测试用例，TEST.md 同步更新，保持文档与代码一致。

/test ./gimp → 重跑并更新 TEST.md

⚠ 使用前须知

  • ·需要 Claude Opus / Sonnet 等前沿模型，小模型生成质量较差
  • ·依赖软件源码，仅有编译二进制的项目覆盖率会下降
  • ·首次生成可能不完整，通常需 /refine 1~3 次达到生产质量
  • ·Windows 需安装 Git for Windows（含 bash + cygpath）或 WSL
